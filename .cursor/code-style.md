# 코드 스타일

## 네이밍 규칙

- **함수/변수명**: camelCase
- **클래스명**: PascalCase
- **인터페이스명**: PascalCase (I 접두사 없음)
- **파일명**: kebab-case
- **상수**: UPPER_SNAKE_CASE

## TypeScript 규칙

- TypeScript 사용 필수
- 명시적 타입 선언 권장
- `any` 사용 지양, `unknown` 사용 권장
- 인터페이스는 `interface` 키워드 사용 (type alias보다 선호)

## 코드 구조

- 한 파일당 하나의 주요 export (클래스/함수)
- 관련된 유틸리티 함수는 같은 파일에 그룹화
- 복잡한 로직은 작은 함수로 분리

## 주석 규칙

- 주석, 테스트명, 설명 등에서 "업계 표준", "현실적인", "일반적인" 등 모호하거나 암시적인 용어를 사용하지 않는다
- 오직 구체적 사실과 동작 결과만을 명확하게 기술한다
